Why Python?

The Python Language was designed to be easy to learn and easy to write. Clean syntax, high-level data structures, dynamic typing, and rich support libraries combine to make Python a very productive tool for many types of programming. Python has displaced C and Java both because it is easier to learn and it is being increasingly used in reasearch laboratories at the universities.

Robots

Robotics is employed to give the students interesting projects to exercise their programming skills, and to give exposure to robots

Programming for robots is very different from programming for other applications. Robots are a part of the physical world. When you tell an on screen turtle to move 100 pixels it moves 100 pixels. When you tell an actual robot car to move 100mm it moves approximately 100mm due to the inaccuracies in construction of the robot and unpredictabiity of the environment. The programmer must rely on sensors to determine where the robot actually is, and the sensors themselves have various inaccuracies and limitations.

Robots are also a real time, event driven application. The robot program cannot ignore an upcoming collision to play a song, an it is annoying if the music has skips and fades to allow the robot to read the collision sensors. Careful planning of the structure of the program is required if everything is to appear to happen all the time.

The first major project in "Think Python" is based around turtle graphics, which was originally designed to be used with physical robots. Robotics provides an interesting and currently exciting application of the Python language.
